# 01 产品信息规则

## 固定元素

`01产品信息.jpg` 中以下内容固定不动：

- 标题 `产品信息`
- 左侧字段名：`名称`、`货号`、`材质`、`颜色`、`尺码`
- 背景与分割结构
- 底部绿线

只替换右侧字段值。

## 字段映射

固定映射如下：

- `名称` 值只能放到名称行。
- `货号` 值只能放到货号行。
- `材质` 值只能放到材质行。
- `颜色` 值只能放到颜色行。
- `尺码` 值只能放到尺码行。

## 颜色格式

- 产品信息页右侧 `颜色` 值必须按单个颜色拆分后用 `/` 连接，例如 `月雾灰/贝壳粉/金穗黄`。
- 优先使用商品 JSON 顶层 `colors` 数组作为颜色顺序；如果没有 `colors`，再读取 `info.颜色`。
- `info.颜色` 中已有的逗号、中文逗号、顿号、斜杠、竖线、分号都视为颜色分隔符，渲染时统一输出为 `/`。
- 不把颜色名合并、不改写颜色名本身，不确定的颜色仍按原识别内容保留。

## 材质换行

- `材质` 字段是否换行由商品 JSON 中 `info.材质` 的显式换行决定。
- 识别或整理 `product_data.json` 时，如果源图材质存在多层结构，例如 `面层/底层`、`面料一/面料二`、`里料/填充物` 等，必须在对应层级之间写入 `\n`。
- 渲染脚本不根据 `面层`、`底层`、`面料一` 等文字标签自动判断换行；普通材质成分中的 `/`、顿号、逗号保持原样。

## 空值

- 如果某字段没有数据，该字段右侧留空。
- 不隐藏该行。
- 不把下面字段上移。
- 不重新排版左侧字段。

## 坐标策略

必须以模板原始行中心为准：

1. 在 2000px 模板中确定每一行右侧值的原始中心点。
2. 清理或覆盖动态值区域。
3. 将新值绘制到同一行中心。
4. 不根据字段是否存在重新计算行间距。

## 长文本

- 优先保持字号稳定。
- 仅当文字超过右侧安全区域时，缩小该字段值。
- 缩小幅度要有限，避免“材质”等字段明显比其他字段小很多。
- 不能为了容纳长文本而移动固定字段名。

## 失败表现

以下都视为不合格：

- 没有材质时颜色/尺码上移。
- 值没有贴合模板原行。
- 固定字段名被重画或错位。
- 字号与模板差异明显。
