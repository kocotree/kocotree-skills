# 批量执行规则

## 目标

从一个输入根目录批量处理全部商品文件夹，按商品分别输出详情页模块。不能写死只处理某几款，不能静默漏款。

## 扫描规则

- 输入根目录下的一级文件夹都视为候选商品。
- 商品文件夹名通常是 `<货号 商品名>`，但不能只靠硬编码货号判断。
- 若某商品缺少某模块素材，只跳过该模块并记录原因；不能跳过整个商品后不说明。
- 输出目录按商品名创建子文件夹，避免不同商品互相覆盖。

## 模板规则

- 默认使用 skill 内置 `template/` 目录。
- 如果用户传入 `--template-dir <template-dir>`，则使用外部模板目录覆盖默认模板。
- 视觉部模板是 2000px 母版，不是 790px 小图模板。
- 所有模块先在 2000px 模板坐标内合成。
- 最终整图按比例缩到 790px 宽。
- 不允许先在 790px 上估坐标再生成，因为会导致错位、白边、圆角丢失。

## 固定元素规则

- 模板已有标题、绿色底、标志、装饰线、卡片圆角、固定文字都应保留。
- 动态内容只覆盖动态区域。
- 表格类模块应整块重建表格区域，不在 JPG 上局部擦除旧线条。
- 任何覆盖层都应在 2000px 阶段贴入，再统一缩放。

## 输出规则

- 输出 JPG。
- 输出文件宽度必须为 790px。
- 成品输出目录只放商品子文件夹和商品图片，不写入报告 JSON。
- 批处理报告统一写入 `scripts/output/report/render_report-<时间戳>.json`。
- 输入商品存在模特图素材时，必须在运行渲染脚本前先执行 AI 审美选图/选框，并输出源图原始像素裁切坐标；脚本不会停下来等待坐标数据。
- AI 模特图选框坐标统一读取 `scripts/output/visual_model_selection/visual_model_selection-<时间戳>.json`，不放入成品目录。
- 本轮生成了 AI 坐标 JSON 时，运行命令必须显式传入 `--visual-model-selection "<坐标json路径>"`，避免误读旧文件或回退默认裁切。
- 目前常规输出：
  - `01产品信息.jpg`
  - `02透明图.jpg`
  - `03选尺码.jpg`
  - `05尺码快选.jpg`
  - `模特图1.jpg`
  - `模特图2.jpg`
  - `模特图3.jpg`
- `04推荐尺码.jpg` 暂不处理，除非用户明确更新规则。

## 防漏检查

执行结束必须列出：

- 扫描到的商品数。
- 成功输出的商品数。
- 每个未处理商品或未处理模块的原因。
- 若商品有模特图素材，说明是否使用 AI 审美选框；未使用时说明原因。
- 对含模特图且已生成坐标的商品，报告中对应 `model_images.*.mode` 必须是 `visual_model_selection`；若出现 `cover_crop`，逐项说明回退原因。
- 输出目录位置。
- 报告 JSON 位置。

## 禁止事项

- 禁止写死只处理某几个商品。
- 禁止写死本机路径、桌面路径、用户目录或项目临时目录。
- 禁止把商品识别结果写死在 Python 渲染脚本里。
- 禁止没有数据时自动补默认值。
- 禁止用旧模板局部擦白后直接叠新表格，容易产生白框和绿线残留。
- 禁止让动态区域覆盖固定模板文字。
- 禁止输出中混入上一次运行的旧结果而不说明。
