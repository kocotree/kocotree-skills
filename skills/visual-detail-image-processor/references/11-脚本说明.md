# 脚本说明

## 现有脚本

```text
scripts/detail_template_replacer_2000.py
scripts/main.py
scripts/create_product_data_template.py
scripts/validate_product_data.py
```

## 脚本运行环境

使用 `uv` 执行脚本。依赖由 skill 根目录的 `pyproject.toml` 管理。

当前外部依赖只有：

```text
Pillow
```

以下都是标准库，不要写入依赖：

```text
argparse
json
pathlib
dataclasses
shutil
typing
copy
```

## 使用原则

- 所有路径都通过参数传入。
- 不写死桌面路径、本机用户目录、货号或商品名。
- 不把商品表格数据写进 Python 脚本。
- 商品数据优先来自 `scripts/output/product_data/<商品文件夹名>-product_data-<时间戳>.json`，默认取最新版本；也兼容商品目录内的 `product_data.json` 或 `--data-dir` 下的 JSON。
- 模特图智能审美裁切坐标来自 `scripts/output/visual_model_selection/visual_model_selection-<时间戳>.json`，也可通过 `--visual-model-selection` 指定。
- 若新增通用能力，应放进 `detail_template_replacer_2000.py`。

## 渲染命令

默认使用 skill 内置 `template/`：

```powershell
cd scripts
uv run python main.py `
  --input-root "<input-root>" `
  --output-root "<output-root>"
```

使用外部数据目录和外部模板目录：

```powershell
cd scripts
uv run python main.py `
  --input-root "<input-root>" `
  --output-root "<output-root>" `
  --data-dir "<data-dir>" `
  --template-dir "<custom-template-dir>" `
  --visual-model-selection "<visual_model_selection-json>"
```

未传 `--data-dir` 时，脚本默认读取 `scripts/output/product_data/` 中每个商品最新的 `*-product_data-<时间戳>.json`；如果该目录没有对应数据，再兼容读取商品目录内的 `product_data.json`。

未传 `--visual-model-selection` 时，脚本默认读取 `scripts/output/visual_model_selection/` 中最新的 `visual_model_selection-*.json`；如果没有该文件，则模特图使用默认 `cover_crop` 裁切。完整结构见 `references/10-智能模特图选框规则.md`。

渲染完成后，成品图片只写入 `--output-root/<商品文件夹名>/`。批处理报告不跟随 `--output-root`，统一写入：

```text
scripts/output/report/render_report-<时间戳>.json
```


## 修改脚本后的检查

每次修改后至少检查：

- 能扫描输入根目录全部商品。
- 输出宽度为 790px。
- 没有硬编码本机路径。
- 没有硬编码商品表格数据。
- 空字段不移动布局。
- 表格无白框、无绿线、无残影。
- 透明图同页商品大小一致。
- 模特图按 `references/09-模特图规则.md` 中的 2000px 坐标框 cover-crop 填满，不留白、不拉伸。
- 没有静默跳过商品。
