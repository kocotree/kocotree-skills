# 智能模特图选框规则

## 目标

`visual_model_selection` 用于保存智能模型主观审美选出的模特图裁切坐标。智能模型只判断保留哪一块画面并输出坐标；脚本只按坐标裁切、缩放和贴入模板，不生成、不扩图、不重画图片内容。

## 存放位置

批次级坐标文件统一放在：

```text
scripts/output/visual_model_selection/
```

推荐文件名：

```text
visual_model_selection-<时间戳>.json
```

如果未通过命令行指定文件，脚本会自动读取该目录下最新的 `visual_model_selection-*.json`。如果目录或文件不存在，脚本回退到原有 `cover_crop` 裁切。

## JSON 数据结构

一个批次使用一个 JSON，包含多个产品和多张模特图：

```json
{
  "version": 1,
  "created_at": "2026-06-17 17:00:00",
  "input_root": "E:/桌面/4",
  "products": {
    "KQ26146 小风迹棉感速干T": {
      "模特图1.jpg": {
        "source": "模特1.jpg",
        "crop_box": [120, 80, 3820, 5290],
        "reason": "头部完整，衣服主体居中，动作自然"
      }
    }
  }
}
```

字段说明：

- `source`：商品目录内的源模特图文件名。
- `crop_box`：源图原始像素坐标，格式为 `[left, top, right, bottom]`。
- `reason`：智能模型选择该画面的审美原因，用于复核，不参与渲染。

## 裁切规则

- 坐标优先级高于脚本默认 `cover_crop`。
- 坐标超出源图范围时，脚本会先夹到源图边界内。
- 坐标比例与模板照片框比例不一致时，脚本以坐标中心为准自动微调到模板比例。
- 坐标无效、缺失或源文件不存在时，该张图回退到默认 `cover_crop`，并写入报告。
- 裁切后仍必须等比例缩放填满模板照片框，不允许拉伸、不允许留白。

## 审美优先级

```text
头部完整 > 脸部和上半身自然 > 服装主体完整 > 动作轮廓完整 > 画面重心舒服 > 和模板相框协调 > 脚部完整
```

横向动作优先保留手臂和服装轮廓；站姿图优先保头肩、衣服主体和自然留白。
