## 设计方向

Kocotree 软件 UI 面向需要长时间使用的内部工具和桌面客户端。整体气质为专业、清楚、克制，并在小面积细节中保留探索感。

遵循以下原则：

- 白色工作面承担主要内容。
- 细边框和浅灰分区表达层级。
- 品牌绿只突出关键操作和当前状态。
- 信息密度偏紧凑，但不牺牲可读性和点击区域。
- 圆形表达亲和，60° 切角表达探索；二者只作为低强度识别细节。
- 页面标题和工作区不使用大型云朵、树冠或密集品牌纹样。
- 软件界面不模仿包装、工牌、文化墙等实体应用的排版。

## 设计令牌

### 色彩

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 品牌绿 | `#009139` | 默认主按钮、选中、Focus、品牌识别 |
| 柔和交互绿 | `#2F8F68` | 明确要求降低视觉强度时替代品牌绿 |
| 墨黑 | `#221814` | 页面标题、重点文字 |
| 暖白 | `#F8F8F1` | 空状态、局部柔和背景 |
| 浅蓝灰 | `#B0CED5` | 信息状态、辅助背景 |
| 珊瑚红 | `#E16B51` | 错误、危险操作、未读提醒 |
| 探索黄 | `#EDDC53` | 警告、需要确认的风险 |
| 页面文字 | `#292B28` | 正文与控件文字 |
| 次要文字 | `#757A75` | 描述、时间、辅助信息 |
| 弱文字 | `#9BA09B` | 计数、占位和非关键注释 |
| 边框 | `#DFE3DF` | 面板、输入框和分隔线 |
| 弱边框 | `#ECEFEC` | 面板内部的轻分隔 |
| 浅背景 | `#F7F8F6` | Hover、分区和次级操作背景 |

主色策略：

- 默认选择 `#009139`。
- 柔和方案使用 `#2F8F68`，且必须替换整个界面的主交互色。
- 同一页面不把标准绿和柔和绿作为两个同等级主色。
- 大面积背景保持白色或浅灰，不铺满品牌绿。

### 字体

使用系统 UI 字体，保证 Windows、macOS 和 WebView 稳定显示：

```css
font-family: Inter, ui-sans-serif, -apple-system, BlinkMacSystemFont,
  "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif;
```

代码、版本号、内部名称和路径使用：

```css
font-family: "SFMono-Regular", Consolas, monospace;
```

推荐层级：

| 层级 | 字号/行高 | 用途 |
| --- | --- | --- |
| 页面标题 | `23/31` | 页面唯一主标题 |
| 面板标题 | `15/22` | 内容分区与组件名称 |
| 卡片标题 | `14/20` | Skill 展示名称 |
| 正文 | `12–14/20` | 描述和表单内容 |
| 控件文字 | `13/20` | 按钮、输入框和下拉触发器 |
| 辅助信息 | `11/16` | 内部名称、版本、计数和帮助文本 |

标题使用 `600–700`，正文使用 `400`，避免在大面积内容中使用极粗字重。

普通说明、帮助文本和状态文字的最小字号为 `11px`。头像缩写、纯装饰编号和通知计数可使用 `9–10px`，但不得承载需要阅读的完整信息。

### 间距、圆角与阴影

- 以 4px 为间距基数，优先使用 4、8、12、16、20、24、32。
- 徽标圆角：6px。
- 输入框、按钮和导航项圆角：8px。
- 面板与卡片圆角：12px。
- 模态框圆角：14px。
- 普通面板不使用明显阴影。
- 弹层、下拉菜单和 Toast 可使用轻阴影，阴影颜色基于墨黑且透明度低于 20%。

## 页面布局

### 应用外壳

- 左侧导航固定宽度，主内容区域自适应。
- 页面内容在宽屏中限制最大宽度并水平居中。
- 页面主体独立滚动，应用外壳不产生额外纵向滚动。
- 最小宽度附近优先重排为两列或一列，不让内容突然横向跳动。

### 页面标题

- 每页只使用一个 `h1`。
- 标题下不放无操作价值的宣传句。
- 页头保持简洁，不使用大型绿色树冠或云朵图形。
- 操作按钮放在标题右侧或内容区域底部，避免两个位置同时重复。

### 内容面板

- 使用 1px 细边框区分区域。
- 面板内部采用 16–20px 内边距。
- 同类面板保持相同标题高度和内容起点。
- 卡片网格 Hover 上移最多 1px，外层必须预留空间避免裁切。

## 组件规范

### 按钮

- 主按钮：品牌绿背景、白色文字、8px 圆角。
- 按钮文字使用 `13px`，保持单行并预留 Loading 状态宽度。
- 次按钮：白色背景、灰色边框、深色文字。
- 幽灵按钮：透明背景，Hover 使用浅灰背景。
- 危险按钮：浅珊瑚背景和珊瑚色文字，不使用品牌绿。
- 图标按钮保持正方形，至少提供 `aria-label`。
- 同一操作区只放一个主按钮。
- Disabled 降低透明度并取消点击反馈。
- Loading 保持按钮宽度稳定，文字不互相挤压。

### 输入框

- 高度默认 38px，紧凑模式 32px。
- 输入内容使用 `13px`，字段名称使用 `12px`，帮助与错误文字不低于 `11px`。
- 左右内边距 11–12px。
- Focus 使用主色边框和低透明度外环。
- 错误状态使用珊瑚色边框，并在输入框下方给出具体修正方式。
- 帮助文本说明用户能理解的作用，不描述内部实现。

### 下拉选择

- 收起态与输入框等高、等圆角、等边框。
- 触发器文字使用 `13px`，选项名称使用 `12px`，用途说明使用 `11px`。
- 右侧使用简洁折线箭头，展开时旋转方向。
- 菜单与触发器相隔 6px，使用 10px 圆角和轻阴影。
- 每个选项可包含名称与一行用途说明。
- 选中项使用浅绿色背景、深绿色文字和右侧勾选标记。
- Hover 使用浅灰背景，不与选中状态混淆。
- 支持方向键、Enter、`Esc`、点击外部关闭，并同步 `aria-expanded` 与 `aria-selected`。

### Tooltip

- Tooltip 用于解释无文字图标、展示被截断的完整文本，或补充一个非关键的简短说明。
- Tooltip 只承载纯文本，不放按钮、链接、表单或需要持续阅读的复杂内容；复杂交互使用 Popover。
- 关键操作说明、错误原因和完成任务所必需的信息必须直接显示在界面中，不能只放在 Tooltip 内。
- 鼠标悬停延迟 `300–500ms` 显示，键盘 Focus 时立即显示；移出、失焦或按 `Esc` 时关闭。
- 默认显示在目标上方，与目标间距 `6–8px`；空间不足时自动翻转，并与视口边缘保持至少 `8px` 间距。
- 使用墨黑或深灰背景、白色文字、`6px` 圆角和轻阴影；文字使用 `11–12px`、最大宽度 `240px`，避免长段落。
- 触发元素保留自身可访问名称；图标按钮继续使用 `aria-label`，Tooltip 通过 `role="tooltip"` 和 `aria-describedby` 提供补充描述。
- Tooltip 不获取焦点，鼠标不可进入；触屏设备不得依赖 Tooltip 承载关键信息。
- Disabled 控件无法获得 Focus 时，应在可聚焦的外层元素上触发说明，或把不可用原因直接写在控件附近。

### 标签与状态

- 可选择 Tag 使用胶囊形按钮，选中后显示浅绿背景。
- Tag 与状态文字使用 `11px`，避免缩成难以阅读的装饰标签。
- 创建 Tag 使用圆形加号按钮，紧跟在已有 Tag 后。
- 状态徽标使用圆点加文字，不能只靠颜色表达。
- 已安装使用绿色，有新版本使用蓝灰，本地修改使用黄色，归档使用中性灰。

### Skill 卡片

- 信息顺序为头像、展示名称、`skillName`、展示简介、上传者、安装量和操作按钮。
- 展示名称、内部名称和简介垂直排列，保持一致节奏。
- 头像与标题只在第一行建立关系，不让内部名称与描述产生过大间隔。
- 卡片整体可点击时，内部操作必须阻止冒泡。
- 已安装状态使用勾选按钮，未安装使用主色加号按钮。

### 人员与账户菜单

- 用户头像、姓名和部门来自身份系统，不提供本地编辑入口。
- 部门路径使用中点或空格分隔，超长时省略。
- 协作者只展示头像，按姓名排序；详情中可通过辅助信息查看姓名。
- 账户菜单位于用户区域正上方并根据用户区域宽度居中。
- 退出登录是菜单项，不让点击用户区域直接退出。
- 未读通知使用小红点，不进行强提醒。

### 反馈、模态框和 Toast

- 成功、警告和错误反馈写明结果与下一步。
- 警告使用浅黄色，错误使用浅珊瑚色，成功使用浅绿色。
- 模态框在窗口中固定居中，头部、内容和底部操作区保持独立。
- 模态框内容区域滚动，标题和底部按钮保持可见。
- 底部操作从右到左排列主按钮、次按钮。
- Toast 默认出现在右下角，2–3 秒后自动收起，并允许手动关闭。

### 空状态

- 说明当前为空的原因和一个明确操作。
- 可在角落使用低透明度树冠圆形作为辅助图形。
- 辅助图形不能遮挡文字、按钮或成为页面主视觉。

## 交互与动效

- Hover 过渡为 120–160ms。
- 模态框和 Toast 进入动画为 140–180ms。
- 按钮按下最多下移 1px。
- 不使用持续循环动画和无业务意义的漂浮效果。
- 遵循 `prefers-reduced-motion`，关闭非必要动画。
- Loading 文案单行显示并预留足够宽度。

## 响应式与无障碍

- 宽屏优先使用两列或三列内容网格。
- 900px 左右开始减少辅助信息并调整网格。
- 720px 左右改为单列内容。
- 320px 宽度下不得出现横向页面滚动。
- 所有交互使用原生按钮、输入框或补全正确 ARIA 角色。
- Focus 必须清晰可见。
- 普通可读文字不得低于 `11px`，且不能仅依靠提高字重弥补字号不足。
- 模态框、下拉菜单和账户菜单支持 `Esc` 关闭。
- 图标按钮提供可读名称，状态同时使用形状或文字。
- 正文和辅助文字满足实际背景下的对比度要求。

## 验收清单

- 页面只有一个明确主标题和一个主要操作。
- 主色策略在整个页面保持一致。
- 普通面板没有明显阴影或大面积彩色背景。
- 页头和主工作区没有大型绿色云朵或树冠图形。
- 表单默认、Focus、错误和 Disabled 状态完整。
- 普通文字最小字号达到 `11px`，正文和控件字号符合层级表。
- 自定义下拉框支持鼠标和键盘操作。
- Tooltip 可通过鼠标悬停和键盘 Focus 触发，按 `Esc` 可关闭，且不会溢出视口。
- 卡片 Hover 不被相邻容器裁切。
- 模态框在小窗口中保持固定且内容可滚动。
- 账户气泡与触发区域自适应居中。
- 320px 至宽屏范围内没有遮挡、跳变和横向溢出。
- 主要流程使用真实内容而非占位符验证。
- 构建、类型检查和现有测试通过。
