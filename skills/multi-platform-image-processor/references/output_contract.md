# 输出契约

## 输出根目录

默认输出目录包含以下平台目录：

- `天猫通用版`
- `CBME`
- `蜂享家＋爱库存`
- `京东`
- `唯品会`
- `站外通用版`

指定单个平台时，输出该平台及其依赖的天猫母版。

## 失败策略

全流程采用“尽量输出”策略：

- 输入包文件夹结构或透明图脏点检测失败时采用“立即停止”策略：只输出报告，不创建任何平台内容。
- 能处理但不完全确定时输出结果，并写入 `风险`。
- 能处理但不完全符合大小或数量约束时输出结果，并写入 `警告`。
- 透明 PNG 必须使用 `pngquant` 压缩；找不到 `pngquant` 时写入 `失败项`。
- 文件无法打开、无法写出或关键来源缺失时写入 `失败项`，不中断整包处理。
- 站外 SKU 的商品卡片或彩色标签识别失败时直接按原图压缩输出并写入 `风险`；模型调用失败、模型输出宽高比异常或差异验收失败时自动重试一次，第二次仍失败才按原图压缩输出，并在 `风险` 中记录两次失败原因。
- 报告中必须保留每个平台的输出路径和每张图片的处理记录。

## 报告字段

报告文件是 UTF-8 JSON，默认保存到 skill 目录下的 `scripts/output/report/<输出目录名>-<时间戳>-report.json`，报告目录最多保留最近 100 份，使用中文字段：

- `处理配置`：源目录、模板目录、输出目录、平台参数、运行时间。
- `输入包检测`：是否通过、结构问题、透明图脏点、可选目录警告和实际识别目录；检测失败时额外包含 `标准输入结构`。透明图存在脏点时包含 `透明图诊断图` 和 `透明图问题汇总`，每个问题项包含 `可视化诊断图`。Agent 必须完整展示标准结构和所有可视化结果。
- `素材扫描`：识别到的源素材目录、图片数量、缺失目录。
- `平台结果`：每个平台的输出路径、输出图片数量、保留空目录。
- `图片记录`：源文件、输出文件、平台、用途、尺寸、格式、大小、处理动作。
- `Agent复核建议`：需要 Agent 做视觉判断的任务名称、图片路径、原因。
- `警告`：不阻断交付但需要关注的问题。
- `风险`：模型去字失败降级、低置信度或可能不完美的问题。
- `失败项`：无法完成的具体文件或步骤。
- `汇总`：平台数、图片数、Agent复核建议数、警告数、风险数、失败数。

## 命名约定

- 天猫、京东、CBME、唯品会详情页从 `601.jpg` 连续递增。
- 蜂享家＋爱库存详情页从 `详情图-01.jpg` 连续递增。
- 其他图片保留原文件名主干，按目标格式改扩展名。
- 文件名冲突时追加 `_2`、`_3` 等序号。

## 模板目录

模板目录用于两类事情：

- 复制已有平台空目录结构，确保缺素材目录仍被保留。
- 查找站外 `800白底图＋logo` 使用的 `logo3.png`。

默认使用 skill 内置 `template` 目录。模板目录中的空目录使用 `.gitkeep` 占位以便 Git 跟踪；复制模板空目录时忽略 `.gitkeep`。

模板目录不存在时，脚本仍按平台规则创建标准输出目录。

## 临时文件

站外 SKU 模型去字的同尺寸输入画布和临时生成图保存到 `scripts/output/image-without-text-tmp`，最多保留最近 100 张图片。临时图只用于 Agent 复核和问题排查，不写入最终交付目录。
