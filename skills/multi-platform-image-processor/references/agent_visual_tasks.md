# Agent 视觉复核任务

## 默认策略

复杂视觉判断由 Agent 在脚本运行前后完成：

- 脚本负责确定性处理、模型去字、尺寸压缩和结构化报告。
- 脚本在报告中写入 `Agent复核建议`。
- Agent 根据 `Agent复核建议` 查看对应图片并判断是否需要重新运行脚本或调整规则。

## 任务一：详情页模块完整性判断

使用场景：

- 天猫 `790详情页` 生成后。
- 蜂享家＋爱库存详情页重新合成长切片后。

判断目标：

- 是否出现明显切断商品模块、标题、表格、尺码区、卖点区。
- 是否存在异常空白、重复拼接、顺序混乱。
- 是否需要在最终说明中标为风险。

## 任务二：站外 SKU 去文字质量判断

使用场景：

- 生成 `站外通用版\800sku去除文字` 后。

判断目标：

- 逐张以原始尺寸并排查看原图与输出图，不能只看缩略联系图。
- 右侧商品卡片彩色标签中的目标文字是否被去除。
- 彩色标签的颜色、渐变、纹理、轮廓、圆角和弧形是否保持原样。
- 白色商品卡片、商品图、人物、手部、背景、画布边缘和图片四角是否保持原样。
- 是否新增任何色块、边框、图形、文字、涂抹或装饰。
- 模型调用成功不代表视觉验收通过；任何目标文字区域外的肉眼可见变化均判定不合格。

## 任务三：透明图裁边质量判断

使用场景：

- 京东 `透明图`。
- 唯品会 `1200透明图`。

判断目标：

- 京东透明图主体是否在 `800x800` 透明画布内贴合最长边，并由额外 4px 放大产生轻微边缘裁切。
- 唯品会透明图是否先提取主体，等比例放入 `1200x1200` 工作图，再按产品裁切，最后主体额外放大 `10px` 且最终输出仍保持一边为 `1200px`。
- 是否保留透明通道。
- 是否误裁主体或仍保留明显透明空边。
