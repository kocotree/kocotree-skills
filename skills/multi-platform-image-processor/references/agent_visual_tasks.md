# Agent 视觉复核任务

## 默认策略

复杂视觉判断由 Agent 在脚本运行前后完成：

- 脚本负责确定性处理、模型去字、尺寸压缩和结构化报告。
- 脚本在报告中写入 `Agent复核建议`。
- Agent 根据 `Agent复核建议` 查看对应图片并判断是否需要重新运行脚本或调整规则。

## 任务一：详情页模块完整性判断

使用场景：

- 天猫 `790详情页` 生成后。
- 蜂享家＋爱库存详情页重新合成长切片后。

判断目标：

- 是否出现明显切断商品模块、标题、表格、尺码区、卖点区。
- 是否存在异常空白、重复拼接、顺序混乱。
- 是否需要在最终说明中标为风险。

## 任务二：站外 SKU 去文字质量判断

使用场景：

- 生成 `站外通用版\800sku去除文字` 后。

判断目标：

- 右侧绿色弧形底条中的白色文字是否被去除。
- 绿色弧形轮廓是否被破坏。
- 白色产品卡片是否被破坏。
- 背景人物图是否被破坏。
- 是否出现多余绿色矩形或明显涂抹块。

## 任务三：透明图裁边质量判断

使用场景：

- 京东 `透明图`。
- 唯品会 `1200透明图`。

判断目标：

- 京东透明图主体是否在 `800x800` 透明画布内贴合最长边，并由额外 4px 放大产生轻微边缘裁切。
- 唯品会透明图是否先提取主体，等比例放入 `1200x1200` 工作图，再按产品裁切，最后主体额外放大 `10px` 且最终输出仍保持一边为 `1200px`。
- 是否保留透明通道。
- 是否误裁主体或仍保留明显透明空边。
