# crop_box 构图规则

## 定义

`crop_box = [x1, y1, x2, y2]`，表示最终要保留的方形区域。render 会把这个方形区域处理为 `1440x1440` 画布。

模型或 agent 只需要输出这个字段。

## 标注目标

`crop_box` 只服务左侧模特穿着展示。右侧商品贴片由脚本生成，标注时参考模板配置里的 `safe_patch_zone`。

优先级：

```text
服装主体清楚且不被右侧贴片遮挡 > 右侧商品贴片完整展示 > 人物左侧居中 > 头顶与下摆留白 > 四肢/道具完整
```

## 核心构图

- 头顶留白按骨架图的 `body_box.top` 计算，帽子、头发、耳机都算视觉头部；最终画布中 `body_box.top` 到顶部约 `120px`。
- `crop_box` 下边界以 `hip_anchor_y` 到 `garment_box` 下限区间的 `60%` 位置为目标，必须落在 `55%-75%` 范围内。
- `garment_box` 只用于确定裁切下边界的最下限，不能直接作为 `crop_box`。
- `crop_box` 必须完整包含 `head_box`，并在横向完整包含 `garment_box`。
- 人物主体放在左侧展示区的视觉中央，不贴边、不偏到右侧贴片区。
- 服装主体不能进入右侧 `safe_patch_zone`，不能被右侧商品贴片或颜色标签明显遮挡。
- 右侧商品贴片和颜色标签必须完整展示，不能被人物、道具或裁切构图破坏。
- 原图右侧空间不够时，直接把 `crop_box` 整体向右移动到源图外，触发生图补背景；宁愿补背景，也不要让左侧穿着主体被右侧商品标签遮挡。

`crop_box` 是源图坐标，最终会缩放到 `1440x1440`。换算源图留白时可用：

```text
crop_y1 ≈ body_box.top - crop_box 边长 * 120 / 1440
crop_y2 ≈ hip_anchor_y + 0.60 * (garment_box.bottom - hip_anchor_y)
```

`hip_anchor_y` 取骨架 `left_hip/right_hip` 中更低的 y 值。

## 可裁内容

可以裁掉：

- 腿和脚。
- 下半身。
- 手持道具和道具边缘。
- 多余天空、地面和背景。

尽量保留：

- 穿在身上的主要服装区域。
- 影响判断衣服款式的领口、门襟、拉链、衣摆。

## 越界补背景

`crop_box` 可以越出源图边界，用来保证人物和服装裁切位置准确。越界区域由脚本调用生成模型补齐背景；agent 或视觉模型不需要输出补背景方式。

标注时只判断“最终应该裁哪里”，不要为了把框塞回源图而牺牲服装主体位置。补齐后的背景必须延续原图同一个场景，天空渐变、海平线、沙滩/海面透视、清晰度、光照和色温都要连续；不能出现像另一张图拼上去的竖向边界、相框感、墙面感、重复纹理、品红残留或明显接缝，不能改动人物和衣服主体。

## 调整方向

- 人物需要更靠左：`crop_box` 向右移动。
- 人物需要更靠右：`crop_box` 向左移动。
- 人物需要更大：缩小 `crop_box`。
- 人物需要更小或避让贴片：放大 `crop_box`。
- 需要更多上方空间：`crop_box` 向上移动。
- 需要更多下方空间：`crop_box` 向下移动。

脚本不会自动执行这些调整；agent 或视觉模型根据检查图修改 `annotations.json`。
