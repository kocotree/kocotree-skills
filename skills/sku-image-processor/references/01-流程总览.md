# 流程总览

## 目标

生成服装类目 `1440x1440` SKU 图。输入为右侧贴片模板、每个颜色的模特图、每个颜色的透明商品图。输出只保留最终产物、标注检查图片、裁切检查图片、标注清单和汇总文件。

## 固定流程

```text
prep -> annotate -> validate-annotation -> render -> review
```

## 输入结构

支持总目录输入：

```text
<input-root>/
  模板.png
  <商品名>/
    模特图/
      <颜色名>.jpg|jpeg|png
    透明图/
      <颜色名>.png
    场景图/
```

也支持单商品目录输入：

```text
<商品名>/
  模特图/
    <颜色名>.jpg|jpeg|png
  透明图/
    <颜色名>.png
  场景图/
```

`场景图/` 仅用于透明抠像模特图。颜色匹配规则见 [素材匹配](02-素材匹配.md)。

## 输出结构

```text
scripts/output/<YYYYMMDD-HHMMSS>_<输入目录名>/
  final/
    <商品名>/
      <颜色名>.jpg|png
  work/
    annotations.json
    summary.json
    annotated/
      _template_regions.jpg
      _final_contact_sheet.jpg
      <商品名>/
        <颜色名>_grid.jpg
        <颜色名>_skeleton.jpg
        <颜色名>_fill_input.jpg
        <颜色名>_fill_result.jpg
    crops/
      <商品名>/
        <颜色名>_crop.jpg
```

目录含义：

- `final/`：最终可交付图片。
- `work/annotated/`：模板参考线、网格、叠加红色裁切框的骨架图、生图补齐输入/结果和成品拼接示例等检查图。
- `work/crops/`：按最终 `crop_box` 得到的裁切检查图，越界时包含生成模型补齐的背景。
- `work/annotations.json`：视觉标注清单。
- `work/summary.json`：各阶段摘要、失败项和关键路径。

内部批次固定写入 `scripts/output/`。`render` 必须提供 `--output <目录>`，脚本把 `final/` 中的最终图片复制到该交付目录，过程文件只保存在批次 `work/` 中。

交付目录必须不存在或为空。任务结束时只向用户展示交付目录和最终图片，批次目录、`work/` 路径和内部检查图仅供 agent 验收与排错。

`scripts/output/` 最多保留 50 个批次。每次 `prep` 成功后按批次时间删除最早目录，直到恢复为 50 个批次，并把清理结果写入本轮 `work/summary.json`。

## prep

脚本自动完成：

1. 扫描输入目录，确认模板、商品目录、`模特图/`、`透明图/`、`场景图/`。
2. 按颜色名匹配模特图和透明商品图。
3. 判定每张模特图为 `opaque`、`transparent` 或 `no_model`。
4. 读取模板和同名 layout 配置。
5. 生成 `work/annotations.json` 空标注骨架。
6. 生成 `work/annotated/_template_regions.jpg`。
7. 生成 `work/annotated/<商品>/<颜色>_grid.jpg`。
8. 运行 YOLO Pose，生成 `work/annotated/<商品>/<颜色>_skeleton.jpg`。
9. 更新 `work/summary.json`。

命令：

```powershell
cd scripts
uv run python main.py prep --input-root "<input-root>"
```

## annotate

脚本把待标注项写入 `work/summary.json`，agent 或视觉模型执行视觉判断并填写 `work/annotations.json`：

1. 先看 `work/annotated/_template_regions.jpg`，确认右侧贴片危险区 `safe_patch_zone`。
2. 再看 `work/annotated/<商品>/<颜色>_grid.jpg`，按源图像素坐标判断保留区域。
3. 参考 `work/annotated/<商品>/<颜色>_skeleton.jpg` 判断人体中心、头部、肩髋范围和姿势类型。
4. 填写 `crop_box`。

命令：

```powershell
uv run python main.py annotate --annotations "output\<批次>\work\annotations.json"
```

## validate-annotation

脚本自动校验已填写的标注，结果写入 `work/summary.json`：

- 坐标格式是否正确。
- `crop_box` 是否为方形。
- `crop_box` 是否与源图有重叠区域。

命令：

```powershell
uv run python main.py validate-annotation --annotations "output\<批次>\work\annotations.json"
```

## render

脚本自动合成与验收：

1. 按标注裁切模特图；`opaque` 原样裁切，`transparent` 先合成场景图，`crop_box` 越界时调用生成模型补齐背景。
2. 在 `work/annotated/*_skeleton.jpg` 上叠加红色裁切框，保存裁切检查图到 `work/crops/`。
3. 按模板配置自动生成右侧商品贴片与颜色名。
4. `crop_box` 越界时保存生图补齐输入和结果到 `work/annotated/`。
5. `pass` 输出到 `final/`，`fail` 不输出最终图。
6. 生成 `work/annotated/_final_contact_sheet.jpg`。
7. 更新 `work/summary.json`。
8. 按必填的 `--output` 把 `final/` 图片复制到外部交付目录。

命令：

```powershell
uv run python main.py render `
  --input-root "<input-root>" `
  --annotations "output\<批次>\work\annotations.json" `
  --output "<交付目录>"
```

## review

脚本从 `work/summary.json` 汇总失败项，agent 根据 `work/annotated/` 和 `work/crops/` 修正标注。

命令：

```powershell
uv run python main.py review --annotations "output\<批次>\work\annotations.json"
```

## 右侧贴片规则

右侧商品贴片由脚本根据模板配置自动生成：透明商品图按颜色名匹配后等比放入 `product_box`，颜色名写入 `text_box`；agent 或视觉模型只需在标注 `crop_box` 时避让 `safe_patch_zone`。

## 模特图处理硬规则

- `crop_box` 必须由 agent 或视觉模型看图判断，不能用人体框、透明区域框或 YOLO 框直接替代。
- 裁切后的服装主体应清楚，并尽量避开 `safe_patch_zone`。
- 最终画布中 `body_box.top` 距离顶部约 `120px`；衣服下摆参考髋部关键点向下多留一点空间。
- 人物需要保持在左侧展示区的视觉中央，右侧商品贴片和颜色标签需要完整展示。
- `crop_box` 可以越界，脚本负责补齐背景缺口。
- 腿、下半身和道具可以裁。
