# 素材匹配

## 颜色名来源

颜色名来自文件名 stem，不识别、不改写、不翻译。

示例：

```text
模特图/浅雾卡.png
透明图/浅雾卡.png
```

对应颜色名为：

```text
浅雾卡
```

## 匹配集合

脚本应取颜色名并集：

```text
所有颜色名 = 模特图颜色名 ∪ 透明图颜色名
```

然后逐个颜色判断输出分支。

## 输出分支

### 有模特图 + 有透明图

输出完整 SKU 图：

```text
final/<颜色名>.jpg
```

处理：

- 模特图按视觉配置作为底图。
- 右侧商品贴片由脚本按模板配置自动生成。
- 必须生成 `final/`、`work/annotated/`、`work/crops/` 和 `work/summary.json`。

### 只有模特图

输出纯模特图：

```text
final/<颜色名>.jpg
```

处理：

- 只按模特图摆放规则输出 `1440x1440`。
- 不使用模板。
- 不添加右侧贴片。
- 不写颜色标签。
- 仍需检查裁切是否合理。

### 只有透明图

输出模板商品图：

```text
final/<颜色名>.png
```

处理：

- 不放模特图。
- 右侧商品贴片由脚本按模板配置自动生成。
- 输出 PNG。

## 跳过规则

- 没有模板且需要模板分支时，不输出，写入报告。
- 同色素材读取失败时，不输出该颜色，写入报告。
- 文件名为空、重复或无法区分颜色时，写入报告并等待人工确认。

## 批量规则

- 每个商品目录独立输出子目录。
- 有几组颜色素材就处理几组，不只处理一张。
- 不把某个颜色的素材拿去补另一个颜色。
