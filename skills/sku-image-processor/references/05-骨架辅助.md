# 骨架辅助

## 目标

YOLO 骨架辅助属于 `prep` 预处理产物，和模板参考线、坐标网格图一起为后续标注服务。它负责生成可检查的人体关键点、人体框、头部框和服装参考框。

最终 `annotations.json` 只填写 agent 或视觉模型确认后的 `crop_box`。

## 生成时机

运行 `prep` 时自动生成：

```powershell
cd scripts
uv run python main.py prep --input-root "<input-root>"
```

可选参数：

- `--pose-model-name`：YOLO pose 权重名或本地权重路径，默认 `models/yolo11n-pose.pt`。
- `--pose-confidence`：人体检测置信度阈值，默认 `0.25`。
- `--skip-pose`：仅在排查依赖或模型问题时跳过骨架辅助。

## 输出结构

```text
scripts/output/<批次>/work/annotated/
  <商品名>/
    <颜色名>_skeleton.jpg
```

文件说明：

- `<颜色名>_skeleton.jpg`：YOLO 骨架、`body_box`、`head_box`、`garment_box` 的可视化检查图；render 后叠加红色 `crop_box`。
- 关键点数量、识别人数、`body_box.top`、`left_hip`、`right_hip`、`hip_anchor_y` 和草稿摘要写入 `summary.json`。

## 使用原则

1. 先看 `<颜色名>_skeleton.jpg`，确认 YOLO 是否抓到正确人物。
2. 再看同目录的 `<颜色名>_grid.jpg`，按最终 SKU 版式重新判断 `crop_box`。
3. 骨架图只用于辅助判断人物和服装大致位置，不能直接作为最终裁切结果。
4. 多人图或误检图必须选择实际穿着商品的主体人物。
5. `crop_box` 必须结合模板 `safe_patch_zone` 避让要求，不能只根据人体外接框生成。
6. 头顶留白以 `body_box.top` 为参照，因为帽子、头发、耳机都属于视觉头部。
7. `crop_box` 下边界以髋点至 `garment_box` 下限区间的 `60%` 位置为目标，必须落在 `55%-75%` 范围内。
8. `garment_box` 只用于确定裁切下边界的最下限，不能直接作为 `crop_box`。
9. `crop_box` 必须完整包含 `head_box`，并在横向完整包含 `garment_box`。

`hip` 在 COCO 人体关键点中指髋关节位置，文档统一称为髋点；`hip_anchor_y` 取 `left_hip/right_hip` 中更低的 y 值。

## 质量边界

YOLO 骨架适合辅助判断人体中心、头部位置和肩髋范围；对宽松衣摆、帽檐、衣服边缘、手持道具、遮挡关系的识别不稳定。

因此，骨架辅助流程只负责缩小标注搜索范围和生成可视化证据；最终裁切坐标必须经过业务规则校验。

## 本地模型文件

骨架模型权重统一放在 `scripts/models/` 下：

```text
scripts/models/
  yolo11n-pose.pt
```

该目录属于运行时资源，不提交到 git。若本地缺少权重文件，可重新下载后放入该目录，或通过 `--pose-model-name` 指定其他可用权重路径。
