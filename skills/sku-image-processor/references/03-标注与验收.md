# 服装视觉标注与验收规则

## 目标

模型或 agent 只负责回答一个问题：这张源图应该裁哪里。

右侧商品贴片由脚本根据模板配置自动生成；标注时只需要参考 `safe_patch_zone`，避免把服装主体裁到明显会被右侧贴片遮挡的位置。

## 必看辅助图

每张模特图标注前查看：

```text
work/annotated/_template_regions.jpg
work/annotated/<商品名>/<颜色名>_grid.jpg
work/annotated/<商品名>/<颜色名>_skeleton.jpg
```

含义：

- `_template_regions.jpg`：确认右侧贴片危险区 `safe_patch_zone`。
- `_grid.jpg`：读取源图像素坐标。
- `_skeleton.jpg`：辅助判断人物和服装大致位置；render 后在同一张图上叠加红色 `crop_box`。

## 唯一标注字段

有模特图的颜色只需要填写：

```json
{
  "crop_box": [x1, y1, x2, y2]
}
```

规则：

- 坐标为源图像素坐标。
- `crop_box` 必须是方形。
- `crop_box` 可以越出源图边界，但必须与源图有重叠区域。
- 最终画布中 `body_box.top` 距离顶部约 `120px`，帽子、头发、耳机都按 `body_box` 计算。
- `crop_box` 下边界以 `hip_anchor_y` 到 `garment_box` 下限区间的 `60%` 位置为目标，必须落在 `55%-75%` 范围内。
- `garment_box` 只用于确定裁切下边界的最下限，不能直接作为 `crop_box`。
- `crop_box` 必须完整包含 `head_box`，并在横向完整包含 `garment_box`。
- 人物需要位于左侧展示区的视觉中央。
- 服装主体不能进入右侧 `safe_patch_zone`，不能被右侧商品贴片或颜色标签明显遮挡。
- 右侧商品贴片和颜色标签必须完整展示。
- 手持道具不作为保留目标，必要时可裁掉，优先展示穿着商品主体。
- 原图右侧空间不够时，应让 `crop_box` 越出源图并调用生图补背景，不能为了留在源图内牺牲遮挡规则。
- 不需要输出脸、头、服装、人体、logo、拉链、帽子、道具等额外框。
- 不需要输出姿态类型。
- 不需要输出补背景方式。

## 脚本验收

脚本只做以下硬判断：

- `crop_box` 存在。
- `crop_box` 是四元数字坐标。
- `crop_box` 坐标顺序正确。
- `crop_box` 是方形。
- `crop_box` 与源图有重叠区域。
- 有骨架辅助时，`crop_box` 下边界位于髋点至 `garment_box` 下限区间的 `55%-75%`。
- 有骨架辅助时，`head_box` 和 `garment_box` 不能被裁断，`body_box.top` 必须保留。
- 有骨架辅助时，映射到最终画布的服装参考框不能进入 `safe_patch_zone`。
- 需要右侧贴片的分支成功生成商品图和颜色名。

脚本不判断：

- 人物是否居中。
- 道具是否完整。
- 脸是否好看。
- 服装面积比例。
- 服装与贴片的重叠比例。
- 越界补背景是否延续原图同一个场景。

这些项目通过 render 后的检查图复查：

```text
work/annotated/<商品名>/<颜色名>_skeleton.jpg
work/crops/<商品名>/<颜色名>_crop.jpg
work/annotated/_final_contact_sheet.jpg
```

`crop_box` 越界时，脚本会调用生成模型补齐背景缺口，`work/crops/` 中保存补齐后的最终裁切检查图，`work/annotated/` 中保存对应的 `_fill_input.jpg` 和 `_fill_result.jpg`。
